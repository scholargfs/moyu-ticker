# 摸鱼盯盘浮窗 · 设计方案

> 日期:2026-06-28
> 状态:已确认,待生成实现计划

## 一句话定位

Mac 上一个**透明置顶、可拖动的小浮窗**,实时显示自选 A 股涨跌,带一键老板键和点开看分时图。要求:内存占用小、进程名伪装、扫一眼就走、绝不弹窗打断。

## 目标用户与场景

上班族在工位摸鱼盯盘。核心诉求是**隐蔽**:老板路过看不出破绽,信息一眼可读,一键即可隐藏。

## 技术栈

- **Tauri 2**:Rust 后端 + Web 前端,打包成 macOS `.app`
- 前端:原生 HTML/CSS/JS(不上重框架),保持轻量
- 图表:**ECharts** 画当日分时图(用户指定,要更花哨的视觉效果)
- 数据源:腾讯免费行情接口,无需 API key
  - 实时报价:`https://qt.gtimg.cn/q=<code1>,<code2>,...`(GBK 编码,需解码)
  - 当日分时:`https://web.ifzq.gtimg.cn/appstock/app/minute/query?code=<code>`(JSON)
- 代码格式:`sh000001`(上证指数)、`sz000002`、`sh600519` 等,前缀 `sh`/`sz`

## 架构与模块

```
┌─ Rust 后端 (src-tauri) ────────────────────┐
│  quote.rs   抓实时行情 / GBK 解码 / 字段解析   │
│  minute.rs  抓当日分时 JSON                   │
│  poll.rs    交易时段判断 + 轮询,emit 事件      │
│  hotkey.rs  全局老板键 (toggle 显隐)           │
│  store.rs   自选股列表本地持久化               │
│  window:    无边框 / 透明 / 置顶 / 可拖动       │
└───────────────────────────────────────────┘
            │ invoke / event
┌─ 前端 (src) ───────────────────────────────┐
│  list 视图:自选股 名称/现价/涨跌幅(绿涨红跌)  │
│  chart 视图:点某只股 → ECharts 当日分时图     │
│  settings:增删自选股代码                      │
└───────────────────────────────────────────┘
```

### 模块职责

- **quote.rs**:输入一组股票代码,批量请求腾讯接口,GBK→UTF-8 解码,按 `~` 切分提取字段。
  - 关键字段(下标从 0 起):`[1]` 名称、`[3]` 现价、`[4]` 昨收、`[5]` 今开、`[31]` 涨跌额、`[32]` 涨跌幅(%)。
  - 输出结构:`{ code, name, price, prev_close, change, change_pct }`。
  - 单只解析失败时跳过该只,不影响其它。
- **minute.rs**:输入单只代码,请求分时 JSON,解析出当日逐分钟 `时间/价格/均价` 序列,返回给前端喂 ECharts。
- **poll.rs**:判断当前是否 A 股交易时段(周一至周五 09:30–11:30、13:00–15:00,本地时区);
  - 盘中:每 **3 秒** 抓一次,通过 `quotes-updated` 事件推给前端;
  - 盘后/周末:停止轮询(省电、降暴露),前端展示最后一次快照。
- **hotkey.rs**:注册全局快捷键 **`⌥⌘H`**,toggle 浮窗显示/隐藏。
- **store.rs**:自选股代码列表持久化到本地(Tauri Store 插件),启动读取、增删即存。
- **window 配置**:无边框、透明背景、`alwaysOnTop`、可拖动;`activationPolicy: Accessory`(不进 Dock、不在 ⌘Tab 出现)。

## 数据流

1. 应用启动 → `store.rs` 读取自选股列表 → 前端渲染 list。
2. `poll.rs` 进入轮询:盘中每 3s 调 `quote.rs` 批量抓取 → emit `quotes-updated` → 前端更新现价与涨跌色。
3. 用户点击某只股 → 前端 `invoke fetch_minute(code)` → `minute.rs` 返回分时序列 → 切到 chart 视图,ECharts 渲染当日走势(价格线 + 均价线)。
4. 用户在 settings 增删代码 → `invoke` 写入 `store.rs` → 立即生效到下一轮轮询。

## 隐蔽性设计

- **老板键**:全局 `⌥⌘H` 瞬间隐藏/唤出。
- **鼠标移出自动半透明**:鼠标离开窗口降到约 15% 不透明度,移入恢复。
- **进程名伪装**:打包时 `productName` / 进程名设为中性名(如 `mdworker-helper`),活动监视器里不显眼。
- **不进 Dock / ⌘Tab**:`activationPolicy: Accessory`。
- **绝不弹窗**:任何异常都走静默降级(见下),不打断用户。

## 异常处理

- 接口超时/失败 → 保留上一次数据,角落显示小灰点表示"未更新",不弹窗。
- 单只股解析失败 → 跳过该只,其它正常显示。
- 网络全断 → 静默后台重试,不报错、不弹窗。
- 分时接口失败 → chart 视图显示一行小字"分时数据暂不可用",可点重试。

## A 股习惯约定

- **涨用红色、跌用绿色**(与国内行情软件一致,勿用欧美绿涨红跌)。
- 涨跌幅保留两位小数,带 `+/-` 号。

## 不做(YAGNI)

- 不做价格提醒 / 到价通知
- 不做下单、不做券商账户登录
- 不做历史 K 线 / 多日走势
- 不做多市场(美股/港股)

以上后续有需要再迭代,首版聚焦"轻量隐蔽地看自选股实时涨跌 + 当日分时"。

## 测试要点

- quote.rs:对一段真实 GBK 响应做解码与字段解析的单元测试(含正常 + 缺字段两种)。
- minute.rs:对真实分时 JSON 样本做解析测试。
- poll.rs:交易时段判断的边界测试(09:30 / 11:30 / 13:00 / 15:00 / 周末)。
- 端到端:启动 app,真实抓一次行情确认浮窗显示正确、老板键生效、点开分时出图。
